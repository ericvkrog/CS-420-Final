// Overview

How ChromaCode Works

Color isn't decoration — it's the grammar. Every line is prefixed with a color tag that tells the interpreter exactly what kind of operation follows.

01

Write in Color

Start every line with a color tag — BLUE, GREEN, YELLOW, PURPLE, or RED. The tag defines what the line does before the text is even read.

02

Validate at Parse Time

If you use the wrong color for a construct — say BLUE for a print statement — the parser raises a ColorTypeError immediately, telling you which color to use instead.

03

Execute

The interpreter walks the AST and runs the program. Functions, loops, conditions, and variables all behave as you'd expect — just with color as the structural backbone.

// Reference

Syntax & Semantics

Every line begins with a color tag. The tag defines what kind of operation follows — mismatched tags throw a ColorTypeError at parse time. Blocks are terminated with RED end. If/elif/else chains require no RED.

Color Tag Role Valid Constructs Example
BLUE Variables / Data Assignment BLUE x = 42
GREEN Functions / Output print(), func, return GREEN print("hi")
GREEN func add(a, b)
YELLOW Conditions if, elif, else YELLOW if x > 0
YELLOW elif x == 0
YELLOW else
PURPLE Loops / Repetition for i in a..b, while PURPLE for i in 1..10
PURPLE while count > 0
RED End / Stop end (closes loops and funcs) RED end
Scoping
Lexical. Functions capture their definition environment.
Type System
Inferred. Supports int, float, string, bool, None.
Error Handling
ColorTypeError at parse time for tag mismatches.
Comments
Single-line with //. Can appear inline or on their own line.
Paradigm
Imperative with light functional support (first-class functions, closures).
File Extension
.chroma
// Design Philosophy

Why ChromaCode?

Most programming languages are purely textual. ChromaCode adds a second channel of meaning — color — that runs in parallel with the text. The goal: make the structure of a program visible at a glance, helping new programmers read and write code more intuitively.

BLUE / Variables

Data is Blue

Blue represents calm, stable information. Variables are the data layer — grounded, reliable, the foundation everything else builds on.

GREEN / Functions

Functions are Green

Green means go, grow, produce. Functions create output and define behavior — the active, generative part of a program.

YELLOW / Conditions

Conditions are Yellow

Yellow signals caution and decision. Every branch point is a moment of choice — slow down, evaluate, pick a path.

PURPLE / Loops

Loops are Purple

Purple represents cycles and repetition — a color of depth and rhythm. Loops repeat, iterate, and persist.

RED / End

Stop is Red

Red means stop. Every block has a clear, unmistakable ending — RED end — so the structure of a program is always legible.
